## How to move a module to GCCEDI

1. Copy the file into GCCEDI preserving its path (`R/theme.R` ‚Üí `GCCEDI/R/theme.R` etc.).
2. In GCCEDI's `app.R`, include the CSS and JS:
   ```r
   tags$head(
     tags$link(rel = "stylesheet", href = "fonts.css"),
     tags$link(rel = "stylesheet", href = "design-tokens.css"),
     tags$link(rel = "stylesheet", href = "design-system.css"),
     tags$script(src = "theme.js")
   )
   source("R/theme.R")
   source("R/axis_helpers.R")
   source("R/chart_defaults.R")
   source("R/mod_kpi_card.R")
   ```
3. For the dimension palette, replace `gcceii_dimension_palette()` with a
   GCCEDI-specific wrapper that maps diversification-dimension names to
   the same `gcc_theme$dimension_base` slots.
4. Run `source("R/theme.R"); write_design_tokens_css()` locally whenever
   you edit the theme, and commit the regenerated `www/design-tokens.css`.

## NOT portable (GCCEII-specific, stays here)

| File | Why it is GCCEII-specific |
|------|---------------------------|
| `R/utils.R` | Contains `DIMENSION_COLS`, `DIMENSION_WEIGHTS`, `DIMENSION_ICONS` ‚Äî all keyed to GCCEII's six integration dimensions. GCCEDI will need its own equivalent. |
| `R/data_loading.R` | Loads COINr output specific to GCCEII's pipeline. |
| `R/translations.R` | Contains EII-specific strings. (Portable *pattern* ‚Äî GCCEDI can mirror the structure ‚Äî but not the content.) |
| `R/mod_charts.R` | Current chart helpers hardcoded to EII dimensions. Phase 3 migration will extract the generic bits into portable helpers. |
| `R/mod_landing_page.R` | EII landing copy and quote carousel. |
| `R/mod_metadata.R` | EII methodology content. |
| `www/styles.css` | Existing app styles ‚Äî to be progressively migrated onto tokens in Phase 3. Still GCCEII-specific until then. |
| `www/rtl.css` | RTL overrides tied to the existing shinydashboard layout. Portable in spirit, but contains selectors matching EII markup that may not exist in GCCEDI. |
